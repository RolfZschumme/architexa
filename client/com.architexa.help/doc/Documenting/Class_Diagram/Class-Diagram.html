Documenting with Class Diagrams

When documenting with a class diagram the following tips may be helpful.

  1. Open the classes/methods that are relevant to your concept or sub-concept.
  2. Follow method call or inheritence relationships to add aditional items to the diagram.
  3. Adjust positions of classes and packages so that all items are clearly visible and relatively positioned in a sensible way (extending/implementing classes should be below their parents, method calls should proceed from left to right generally).
  4. Turn on "Toggle Context for Diagram Labels" if it is necessary to remove ambiguity.
    • This option can be accessed by right clicking anywhere on the diagram, selecting "Toggle Context for Diagram Labels", and choosing "Show Context"
  5. Add comments/javadoc to explain any key parts.
  6. Double check that relationships are elegantly arranged and not obscuring information.
