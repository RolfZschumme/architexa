Documenting with Sequence Diagrams

The following tips will help you document code details and logic in a sequence diagram.

  1. If there is a complicated section of relationships or time sensitive portion of code in your class diagram / code concept you may want to expand on it with a sequence diagram.
  2. Open a method call (or collection of classes / method calls)
  3. Follow forward connections to relevant methods.
  4. Show all calls if you want to show the details of a specific method.
  5. Overridden methods can be shown by clicking the small up arrow
  6. Overridden method connections can be shown all the time by right clicking on the connection and clicking show always
  7. Once you have expanded the important methods and are displaying the minimum amount of useful information necessary to convey the concept you should add comments
